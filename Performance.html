# 我们不妨来看一道面试题：从用户敲下地址栏，到页面展现出来，浏览器做了什么？

这是一道很经典的面试题，在我的理解下，分别会做：

  • 检查输入的网址是ip还是域名
  • 若是域名则进行dns查询，查询到对应的主机ip
  • 与主机建立tcp连接，向服务器请求数据
  • 浏览器解析服务端返回的数据，渲染页面

# dns预解析
  1. 何为dns？
  2. dns解析过程

答：

  1. DNS是一个域名系统，可以通过解析域名来获取你要访问的ip地址
  2. dns解析过程：（假设访问www.taobao.com）
    • a. 检查浏览器的dns缓存（第一次访问没有）
    • b. 检查系统的dns缓存（第一次访问没有）
    • c. 检查浏览器的host
    • d. 访问根域名服务器，获取.com顶级域名服务器的地址
    • e. 访问.com顶级域名服务器，获取taobao.com的域名服务器地址
    • f. 访问taobao.com的域名服务器地址，获取www.taobao.com的地址
    • g. 一般上面三点是不需要的，因为我们一般会通过网络运营商的dns服务器，由它们进行代理查询，如果它们服务器内有相应的缓存，则不需要代理查询，直接返回ip

所以什么叫预解析？

  • 预解析就是在页面还未访问相应域名前，对于下个页面可能需要获取的资源域名或者相关页面域名的提前解析，所以我个人认为在大部分中小网站上是用不上的
# 缓存
  1. 缓存分为内存缓存和磁盘缓存，内存缓存则缓存一些小的资源，磁盘缓存则缓存所有资源，浏览器优先从内存缓存读取资源
  2. 所以只对第二次访问网站的用户有效
  3. 网站的缓存策略很重要，每个网站上线前都要做好相应的缓存策略，至于为何，以及缓存的内容，以后专门写一篇文章来细说
# 资源请求和页面渲染过程优化
  1. 所谓渲染过程的优化，则是尽量不做阻塞页面渲染的行为，如在body前引入与页面无关的部分js库、脚本资源等
  2. 做页面预加载 prefetch，资源按需加载等
  3. 图片懒加载
  4. 资源过多的时候可以分多几个域名来获取资源（浏览器每个域名限制了8个tcp连接）
